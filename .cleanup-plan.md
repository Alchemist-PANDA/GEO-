# AI Slop Cleanup Plan

## Scope
- `test_llm.py` - Simple API test script
- `measure_lift.py` - Lift calculation utility

## Behavior to Preserve
- `measure_lift.py`: Calculate lift metrics from pre/post JSON files, write to both `lift_report.json` and `final_lift_report.json`
- `test_llm.py`: Test Gemini API connectivity with simple prompt

## Identified Slop

### measure_lift.py
1. **Duplicate file writes** - Writes same data to both `lift_report.json` and `final_lift_report.json` (lines 37-41)
2. **Hardcoded conclusion** - Always says "Lift observed" even when score decreases (line 34)
3. **Dead logic** - `is_cited_improved` and `is_cited_maintained` calculated but not used in conclusion (lines 29-30)
4. **Missing edge case** - Division by zero handled but returns 0% change, which is misleading when pre_score is 0

### test_llm.py
1. **Outdated model** - Uses `gemini-2.0-flash` which has quota issues, should use `gemini-2.0-flash-lite` (line 10)
2. **Generic error handling** - Catches all exceptions without distinguishing quota vs auth vs network errors (line 14)

## Cleanup Passes

### Pass 1: Dead code deletion
- Remove duplicate file write in `measure_lift.py` (keep only `final_lift_report.json`)

### Pass 2: Logic fixes
- Fix conclusion generation to reflect actual lift direction (positive/negative/none)
- Switch `test_llm.py` to `gemini-2.0-flash-lite`

### Pass 3: Error handling
- Improve error messages in `test_llm.py` to distinguish quota/auth/network errors

## Verification Plan
- Run `measure_lift.py` with existing test data, verify output matches expected format
- Run `test_llm.py` after model switch, verify it connects successfully
- Check that only `final_lift_report.json` is written (not both files)
